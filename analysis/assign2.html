Closing Comments

Make_Monotone

This routine of our implementation has remained unusable despite multiple efforts to debug. The essential error was tracked down to insertion of helper/deletion of edge operations for certain vertices.
We were unable to determine the exact conditions under which the error definitely shows, but it seems that polygons with a lesser number of split/merge vertices are getting monotonized properly.

Triangulate_Monotone

This routine takes in a list of polygons which are assumed to be monotone and proceeds to triangulate them.
Degeneracies:
Floating point error can cause some unexpected results.
Complexity:
The triangulation of a monotone polygon is achieved in linear time. The algorithm used is to make a priority queue based on y-coordinates and then adding diagonals while traversing down the chains.
Robustness:
Since this method depends on the monotone nature of passed input, the first step is to check whether the polygon is really monotone. Only after verification does the rest of the algorithm proceed.
