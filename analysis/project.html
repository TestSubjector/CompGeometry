Project

Given a set of n disjoint segments and a point that does not lie on any of them, find the number of visible segments.

The definition of a visible segment is that if any point q lying on a segment S is visible, then the segment S is visible.

Closing Comments

Sorting the points by polar coordinate

The endpoints of the segments are converted into polar coordinates and then arranged in order of increasing theta. This operation can be done in O(n log n) time

Looping over the event points

Based on the order obtained from the sort, we go over the end points one at a time. At each end point, an active list of segments is maintained. At the starting of a segment, it is added into the list, and at the ending, it is removed from the list.
The segment being added/removed is also checked against the other members of the active list to determine if it is visible. This takes O(n) time. If it is visible, it is added to the visibilty list.
Degeneracies:
In the case that the camera point and the end point of one segment and the start point of another segment are collinear, both vertices are considered to be visible. Since in such a situation the edges are definitely visible, it doesn't cause a difference if we let the vertex remain visible.
Complexity:
Insertion operations while looping over n elements are O(1). Hence, the overall complexity will be O(n).
